允许跨源资源共享 (CORS) 策略对安全性敏感。 它在过去导致了以下漏洞：

  • CVE-2018-0269
  • CVE-2017-14460

推荐的安全编码实践

  • Access-Control-Allow-Origin 标头应该仅为受信任的来源和特定资源设置。

敏感示例

nodejs http built-in module:

const http = require('http');
const srv = http.createServer((req, res) => {
  res.writeHead(200, { 'Access-Control-Allow-Origin': '*' }); // 敏感
  res.end('ok');
});
srv.listen(3000);

Express.js framework with cors middleware:

const app = express();

app.use((req, res, next) => {
    res.header('Access-Control-Allow-Origin', '*'); // Sensitive
    next();
});

合规示例

nodejs http built-in module:

const http = require('http');
const srv = http.createServer((req, res) => {
  res.writeHead(200, { 'Access-Control-Allow-Origin': 'trustedwebsite.com' }); // Compliant
  res.end('ok');
});
srv.listen(3000);

Express.js framework with cors middleware:

const app = express();

app.use((req, res, next) => {
    res.header('Access-Control-Allow-Origin', 'trustedwebsite.com'); // 合规
    next();
});
