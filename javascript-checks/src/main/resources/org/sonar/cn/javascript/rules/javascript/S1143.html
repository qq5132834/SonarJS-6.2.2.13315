使用 finally 块中的 return、break、throw 和 continue 覆盖类似来自挂起的 try 和 catch 块的语句。

当跳转语句break、continue、return 和 throw会强制控制流时， 此规则会引发问题留下一个 finally 块。

不合规示例

function foo() {
    try {
        return 1; // 我们期望返回 1
    } catch(err) {
        return 2; // 在错误的情况下返回 2
    } finally {
        return 3; // 不合规: 3 在 1 或 2 之后返回，这真是匪夷所思
    }
}

合规示例

function foo() {
    try {
        return 1; // 我们期望返回 1
    } catch(err) {
        return 2; // 在错误的情况下返回 2
    }
}
