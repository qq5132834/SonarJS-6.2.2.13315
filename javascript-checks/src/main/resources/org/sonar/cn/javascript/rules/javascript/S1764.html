运算符两边的值相同会产生可预测的结果，应该加以简化。

不合规示例

if (a == b && a == b) { // 如果第一个是真的，那么第二个也是
  doX();
}
if (a > a) { // 总是 false
  doW();
}

var j = 5 / 5; //总是 1
var k = 5 - 5; //总是 0

例外

针对自身测试一个变量的特定情况是对 NaN 的有效测试，因此将被忽略。

类似地，将 1 左移到 1 在位掩码的构造中很常见，但会被忽略。

此外，逗号运算符 , 和 instanceof 运算符将被忽略，因为存在使用有效的用例。

if (f !== f) { // 测试 NaN 值
  console.log("f is NaN");
}

var i = 1 << 1; // 合规
var j = a << a; // 不合规
