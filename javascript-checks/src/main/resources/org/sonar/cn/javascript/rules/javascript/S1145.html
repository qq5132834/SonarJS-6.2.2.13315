if 条件始终为假的语句使代码块无法运行。 if 条件始终为真的语句是完全多余的，并且会降低代码的可读性。

存在此类代码的可能原因有以下三种：

  • 调试期间更改了 if 语句，并且已提交该调试代码。
  • 一些值未设置。
  • 有些逻辑并没有按照程序员的想法去做。

在任何这些情况下，都应删除无条件的 if 语句。

不合规示例

if (true) {  // 不合规
  doSomething();
}
...
if (false) {  // 不合规
  doSomethingElse();
}

if (!options || options === true) { doThirdThing(); }  // 不合规，总是true

合规示例

doSomething();

doThirdThing();
