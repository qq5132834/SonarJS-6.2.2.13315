未使用的参数具有误导性。 无论传递给此类参数的值是什么，行为都是相同的。

不合规示例

function doSomething(a, b) { // "a" 没有被使用
  return compute(b);
}

合规示例

function doSomething(b) {
  return compute(b);
}

例外

在编写函数回调时，一些参数可能需要作为函数签名的一部分，但回调代码实际上并不需要。 例如，JQuery 有 'each' 助手来遍历数组或对象，但使用计数器 'i' 应该保持可选：

$(["first", "last"]).each(function (i, value) {
  computeSomethingWithValue(value);
});

因此，只有在参数列表末尾列出的未使用的参数才会被标记为有问题，因为它们可以从函数签名中省略。 未使用的参数后跟是 使用的参数将被忽略。

例如：

var myFirsCallBackFunction = function (p1, p2, p3, p4) {  // p2 被忽略，但 p4 被报告
                                              return p1 + p3; }

var mySecondCallBackFunction = function (p1, p2, p3, p4) {  // p1、p2 和 p3 被忽略
                                              return p4; }

var myThirdCallBackFunction = function (p1, p2, p3, p4) {  // p1 被忽略但 p3 和 p4 被报告
                                              return p2; }

此外，当在函数体中使用 arguments 时，没有参数被报告为未使用。

function doSomething(a, b, c) {
  compute(arguments);
}
