许多 JavaScript 的 Array 方法返回数组的更改版本，同时保持源数组不变。 reverse 和 sort 不属于这一类。 相反，除了返回更改后的版本之外，它们还更改了源数组，这可能不是预期的结果。

当分配这些方法的返回值时，此规则会引发一个问题，这可能导致维护人员忽略原始值已更改的事实。

不合规示例

var b = a.reverse(); // 不合规
var d = c.sort(); // 不合规

合规示例

var b = [...a].reverse();  // 解构并创建一个新数组，因此反向不会影响 'a'
a.reverse();

c.sort(); // 这对数组进行排序
