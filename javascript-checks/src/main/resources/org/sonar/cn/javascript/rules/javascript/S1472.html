因为语句末尾的分号是可选的，所以在单独的行上开始函数调用参数会使代码混乱。 它可能会导致错误，并且很可能会给维护者带来问题。

容易混淆开发者的真实意图

  1. 定义一个函数然后在闭包中执行一些不相关的代码？
  2. 将第二个函数作为参数传递给第一个函数？

第一个选项将由 JavaScript 解释器选择。

通过扩展，为了提高可读性，任何类型的函数调用参数都不应该从新行开始。

不合规示例

var fn = function () {
  //...
}

(function () { // 不合规
  //...
})();

合规示例

例1

// 定义一个函数
var fn = function () {
  //...
}; // <-- 添加分号

// 然后在闭包中执行一些代码
(function () {
  //...
})();

例2

var fn = function () {
  //...
}(function () { // <-- 在同一行启动函数调用参数
  //...
})();
