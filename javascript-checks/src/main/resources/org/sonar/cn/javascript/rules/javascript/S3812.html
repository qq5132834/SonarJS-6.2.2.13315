混淆操作顺序几乎总是会产生意想不到的结果。

同样，误用否定也会产生不好的结果。 例如，考虑 !key in dict 和 !(key in dict) 之间的区别。 第一个在 dict 中查找布尔值 (!key)，另一个查找字符串并反转结果。 !obj instanceof SomeClass 也有同样的问题。

当 in 或 instanceof 运算符的左操作数被取反时，此规则会引发问题。

不合规示例

if (!"prop" in myObj) {  // 不合规，  in  运算符正在检查属性 false
  doTheThing();  // 这个块将永远不会被执行
}

if (!foo instanceof MyClass) {  // 不合规， "!foo" 返回布尔类型，这不是任何东西的实例
  doTheOtherThing();  //这里不会执行
}

合规示例

if (!("prop" in myObj)) {
  doTheThing();
}

if (!(foo instanceof MyClass)) {
  doTheOtherThing();
}
