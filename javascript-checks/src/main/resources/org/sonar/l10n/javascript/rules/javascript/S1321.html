使用 with 关键字会在 JavaScript 严格模式代码中产生错误。 然而，这还不是针对 with 的最坏情况。

使用 with 允许快速访问对象的属性 - 假设它们已经设置。 但是使用 with 访问一些尚未在对象中设置的属性，突然间你会跳出对象范围并进入全局范围，在那里创建或覆盖变量。 由于 with 的效果完全取决于传递给它的对象，因此 with 可能具有不可预测的危险，因此永远不应使用。

不合规示例

var x = 'a';

var foo = {
  y: 1
}

with (foo) {  // 不合规
  y = 4;  // 修改 foo.y 的值
  x = 3;  // foo.x 属性不存在
}
print(foo.x + " " + x); // 输出: undefined 3

合规示例

var x = 'a';

var foo = {
  y: 1
}

foo.y = 4;
foo.x = 3;

print(foo.x + " " + x); // 输出: 3 a
