== 和 != 运算符在比较值之前进行类型转换。 这很危险，因为它可以掩盖类型错误。 例如，它将 '\t\r\n' == 0 评估为 true。

最好使用无副作用的 === 和 !== 运算符。

不合规示例

if (var == 'howdy') {...} // 不合规

合规示例

if (var === 'howdy') {...}

例外

即使测试变量与 null 的相等性并不完全符合大多数 JavaScript 开发人员的想法，在这种情况下使用 == 或 != 也是可以容忍的。 在下面的例子中，如果 foo 没有被初始化，它的默认值不是 null 而是 undefined。 尽管如此，undefined == null，因此 JavaScript 开发人员得到了预期的行为。 

if(foo == null) {...}
