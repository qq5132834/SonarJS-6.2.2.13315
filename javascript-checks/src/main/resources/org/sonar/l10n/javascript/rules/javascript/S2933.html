readonly 属性只能在类构造函数中或在声明时分配使用。 如果类的属性未标记为readonly，但仅在构造函数中设置，则可能会导致该属性的预期用途模糊不清晰。 为避免混淆，此类属性应标记为readonly，以明确其预期用途，并防止未来维护人员无意中更改其用途。

不合规示例

class Person {
  private _birthYear: number;  // 不合规
  constructor(birthYear: number) {
    this._birthYear = birthYear;
  }
}

合规示例

class Person {
  private readonly _birthYear: number;
  constructor(birthYear: number) {
    this._birthYear = birthYear;
  }
}
